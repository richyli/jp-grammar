# 章末綜合測驗 — 資料格式規格（QUIZ_SCHEMA v1）

每章一個測驗檔 `quiz/mXX_quiz.js`，50 題。檔案第一行就是程式碼（**禁止** markdown fence）：

```js
window.GRAMMAR_QUIZZES = window.GRAMMAR_QUIZZES || [];
window.GRAMMAR_QUIZZES.push({
  moduleId: "m01",           // 對應 data/mXX 的模組 id
  title: "句子的骨架",        // 該章標題（與模組一致）
  questions: [ /* 50 題，見下 */ ]
});
```

## 題目物件

每題都是「選擇題」（4 選 1），可選擇性附一段 `passage`（閱讀題幹）：

```js
{
  level: "N3",                       // JLPT 等級 N5|N4|N3|N2|N1（必填，會顯示徽章）
  category: "は vs が",               // 選填：考點短標，顯示在題號旁
  passage: "新製品[しんせいひん]は…",  // 選填：閱讀題幹（新聞/廣告/告示/會話片段），含振假名
  q: "（　）に入る最も適切なものはどれか。",  // 題幹（可含（　）空格），含振假名
  options: ["は", "が", "を", "に"],  // 4 個選項；日文選項要標振假名
  answer: 1,                          // 正解 index（0 起算）
  explain: "「首相」は文全体の主題なので「は」。……"  // 中文解說，含振假名
}
```

## 出題鐵律

1. **50 題**，題型全為 choice（4 選 1）；其中**至少 15 題**帶 `passage`（閱讀導向：讀一段新聞/廣告/告示/購物/用餐/生活會話後作答）。
2. 每題標 `level`；全章 50 題的等級分布要**貼近該章文法點的實際等級分布**（該章有多少 N2 點就多出點 N2 題），並涵蓋該章從最低到最高的等級。
3. 題目要**覆蓋該章所有主要文法點**，不要集中在少數幾點。可設計「近義文法二選一」的鑑別題（例：は vs が、と vs たら、ため原因 vs 目的）。
4. **場景多樣**：passage 與題幹情境輪流取材——新聞（政治/經濟/災害/社會）、廣告文案、車站/店頭告示、購物對話、餐廳點餐、日常生活會話。不要全是新聞。
5. 解說 `explain` 用**台灣繁體中文**，簡潔點出「為什麼選這個、為什麼不選別的」，1–3 句即可。
6. **振假名**：所有日文（passage、q、options、explain 內引用的日文）漢字一律 `漢字[かな]`；送假名不入括號（読[よ]む）。片假名、平假名不標。
7. 字串內**禁用未跳脫的英文雙引號**；日文引用用「」『』。
8. 選項要有誘答力（似是而非的干擾項），不要明顯錯到不用讀就能刪。
9. `answer` 的正解位置要打散（不要總是同一個 index）。
