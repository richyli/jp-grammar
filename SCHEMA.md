# 日語文法教室 — 資料模組規格（SCHEMA v1）

每個內容模組是一個獨立的 `.js` 檔，放在 `data/` 目錄。檔案**必須**是合法的 JavaScript，
以下列固定樣板開頭與結尾（不得包在 markdown code fence 內，檔案第一行就是程式碼）：

```js
window.GRAMMAR_MODULES = window.GRAMMAR_MODULES || [];
window.GRAMMAR_MODULES.push({
  // ...模組物件（見下方規格）
});
```

## 鐵律

1. **繁體中文（台灣用語）**撰寫所有解說。不用對岸用語（「視頻」「信息」等禁用）。
2. **所有日文例句中的漢字都要標假名**，使用振假名標記：`漢字[かんじ]`（漢字連串後緊接方括號假名）。
   例：`日本[にほん]の新聞[しんぶん]を読[よ]む`。送假名不放進方括號（`読[よ]む`，不是 `読む[よむ]`）。
   片假名、平假名不標。數字+單位如 `3人[にん]` 標單位漢字即可。
3. **字串內不要使用未跳脫的英文雙引號**。日文引用一律用「」『』；需要強調用 <b>。
4. 例句以**新聞、廣告文體為主**（政治、經濟、社會、災害、科技新聞；商品廣告、車廂廣告、店頭 POP、企業標語），
   輔以少量日常會話對照。每個例句標 `source` 欄位。
5. 每個文法點標 `level`（N5/N4/N3/N2/N1），但解說順序照系統邏輯，不照等級。
6. 解說要「解決閱讀問題」導向：先講這個文法在真實文章裡長什麼樣、讀者會在哪裡卡住，再講規則。
7. `explanation` 內可用簡單 HTML：`<b>`、`<br>`、`<span class="jp">日文</span>`。日文段落內同樣要標假名。

## 模組物件規格

```js
{
  id: "m01",              // 模組代號，檔名一致
  order: 1,               // 顯示順序（整數）
  title: "句子的骨架",     // 模組標題（中文）
  subtitle: "語順・判斷句・は與が",  // 副標
  icon: "🦴",             // 一個 emoji
  intro: "…",             // 150-300字模組導言：為什麼讀新聞需要這一章、學完能解決什麼閱讀問題
  estMinutes: 180,        // 預估學習分鐘數（點數×8分鐘左右）
  openingPuzzle: {        // 開場閱讀謎題：一段真實風格的新聞或廣告片段
    source: "新聞",       // "新聞" 或 "廣告"
    jp: "…",              // 日文（含假名標記），2-4句
    zh: "…",              // 中文翻譯
    hook: "…"             // 1-2句：指出這段裡讀者「看不懂的地方」正是本章要解決的
  },
  points: [ /* 文法點陣列，見下 */ ],
  closingDecode: {        // 章末回頭解讀 openingPuzzle
    jp: "…",              // 同一段（或延伸段）日文
    walkthrough: "…"      // 逐句解構說明（中文，可用 <br> 分行），點名用到本章哪些文法點
  },
  finalExercises: [ /* 3-6題綜合練習，格式同 exercises */ ]
}
```

## 文法點（point）規格

```js
{
  id: "m01-wa-topic",     // 全站唯一，模組代號開頭，小寫英數與連字號
  name: "は（主題標記）",   // 文法點名稱：日文文型 +（中文短註）
  level: "N5",            // N5 | N4 | N3 | N2 | N1
  category: "係助詞",      // 分類詞（助詞/句型/文體/敬語/機能語…自由但precise）
  connection: "名詞＋は",  // 接續方式（日文文法術語+記號，可用 "普通形＋…" 等）
  meaning: "標示句子的主題：「至於…」",   // 一句話中文核心義
  explanation: "…",       // 200-450字中文解說。結構：①讀新聞/廣告時哪裡會遇到、②規則本體、③易混淆點或陷阱。可用簡單HTML
  visual: { … },          // 視覺化圖解（必填，型別見下）
  examples: [ … ],        // 至少3個例句
  exercises: [ … ],       // 至少2題
  compare: ["m01-ga-subject"]   // 選填：易混淆點的 point id 陣列（站內會生成連結）
}
```

## 例句（examples 陣列元素）

```js
{
  jp: "政府[せいふ]は新[あたら]しい政策[せいさく]を発表[はっぴょう]した。",
  zh: "政府發表了新政策。",
  source: "news",        // "news"新聞 | "ad"廣告 | "daily"日常 | "notice"告示 | "shop"購物 | "dining"用餐
  note: "新聞正文常用普通形（た形）結尾。"   // 選填：一句閱讀提示
}
```

## 視覺化（visual）— 六種型別擇一

每個文法點都要有 visual。選最能表達該文法本質的型別：

### 1. structure — 句子解剖（最常用：語順、助詞、句型結構）
```js
{ type: "structure", title: "句子成分分解",
  chunks: [
    { jp: "政府[せいふ]", role: "主題", zh: "政府" },
    { jp: "は", role: "助詞", zh: "" },
    { jp: "新[あたら]しい政策[せいさく]", role: "目的語", zh: "新政策" },
    { jp: "を", role: "助詞", zh: "" },
    { jp: "発表[はっぴょう]した", role: "述語", zh: "發表了" }
  ] }
```
role 限用：主題 | 主語 | 目的語 | 補語 | 述語 | 修飾 | 助詞 | 接續 | 引用 | 其他

### 2. timeline — 時間軸（時態、時貌、時間關係機能語）
```js
{ type: "timeline", title: "「ている」的時間結構",
  now: 55,                                  // 「現在」在軸上的位置 0-100，選填
  events: [ { label: "開始[かいし]", pos: 20 }, { label: "持續中", pos: 55 } ],
  spans:  [ { from: 20, to: 80, label: "動作持續" } ] }
```

### 3. flow — 流程/方向圖（授受、被動方向、變化過程、接續邏輯）
```js
{ type: "flow", title: "「てもらう」的恩惠方向",
  steps: [ { jp: "先生[せんせい]", zh: "老師（動作者）" },
           { jp: "→ 教[おし]えてもらう →", zh: "恩惠流向" },
           { jp: "私[わたし]", zh: "我（受益者）" } ] }
```

### 4. table — 對照表（活用表、敬語對照、條件形比較）
```js
{ type: "table", title: "四種條件形比較",
  headers: ["形式", "核心義", "例"],
  rows: [ ["と", "必然/自動", "春[はる]になると桜[さくら]が咲[さ]く"], … ] }
```

### 5. tree — 巢狀結構（連體修飾、長句解剖）
```js
{ type: "tree", title: "連體修飾的巢狀結構",
  root: { label: "ニュース（名詞）", note: "被修飾的核心",
          children: [ { label: "政府[せいふ]が発表[はっぴょう]した", note: "修飾句①" } ] } }
```

### 6. compare — 並列卡片（兩三個易混文法對比）
```js
{ type: "compare", title: "「は」vs「が」",
  columns: [
    { header: "は", cells: ["舊資訊・主題", "焦點在後面", "象[ぞう]は鼻[はな]が長[なが]い"] },
    { header: "が", cells: ["新資訊・主語", "焦點在前面", "誰[だれ]が来[き]たの？"] } ] }
```

## 練習題（exercises / finalExercises 陣列元素）— 三種型別

### choice — 選擇題（題幹可含（　）空格）
```js
{ type: "choice",
  q: "首相[しゅしょう]（　）記者会見[きしゃかいけん]で語[かた]った。",
  options: ["は", "を", "に", "で"],
  answer: 0,              // 正解 index（從0起算）
  explain: "「首相」是這句的主題，後面整句都在講首相做了什麼，用「は」。" }
```

### cloze — 克漏字（多個空格版的閱讀題，題幹是一小段文章）
```js
{ type: "cloze",
  passage: "新製品[しんせいひん]（　①　）来月[らいげつ]発売[はつばい]（　②　）れる。",
  blanks: [
    { options: ["が", "は", "を"], answer: 1, explain: "…" },
    { options: ["さ", "ら", "し"], answer: 0, explain: "…" } ] }
```

### reading — 閱讀理解（一段新聞/廣告 + 1-3個理解問題）
```js
{ type: "reading",
  passage: "台風[たいふう]10号[ごう]の接近[せっきん]に伴[ともな]い、明日[あす]の便[びん]は欠航[けっこう]となる恐[おそ]れがあります。",
  passageZh: "隨著十號颱風接近，明天的航班恐將停飛。",   // 作答後才顯示
  questions: [
    { q: "「恐れがあります」表示什麼？",
      options: ["已經決定停飛", "有停飛的可能性", "絕對不會停飛"],
      answer: 1, explain: "「恐れがある」＝有…的（負面）可能性，新聞高頻。" } ] }
```

## 完整文法點範例（照這個密度寫）

```js
{
  id: "m08-osore",
  name: "恐れがある（恐將…）",
  level: "N2",
  category: "推測・新聞文體",
  connection: "動詞辭書形／名詞の＋恐れがある",
  meaning: "有發生（壞事）的可能性；新聞預警高頻句型",
  explanation: "看日本新聞，只要出現災害、事故、經濟惡化的預測，幾乎都會看到「〜恐[おそ]れがある」。它表示<b>負面事態發生的可能性</b>，語氣比「かもしれない」正式，且只用於壞事——你不會看到「合格[ごうかく]する恐れがある」。<br>閱讀時的判斷關鍵：句尾看到「恐れ」就知道①這件事還沒發生、②寫的人認為它是壞事、③這是預測不是事實。標題裡常縮成名詞止め：「大雨[おおあめ]の恐れ」。",
  visual: { type: "structure", title: "新聞預警句的結構",
    chunks: [
      { jp: "台風[たいふう]の接近[せっきん]に伴[ともな]い", role: "接續", zh: "隨著颱風接近" },
      { jp: "交通[こうつう]が乱[みだ]れる", role: "修飾", zh: "交通混亂（還沒發生）" },
      { jp: "恐[おそ]れがある", role: "述語", zh: "恐將／有…之虞" } ] },
  examples: [
    { jp: "今夜[こんや]から明日[あす]にかけて、土砂災害[どしゃさいがい]の恐[おそ]れがあります。", zh: "今晚到明天，恐有土石流災害。", source: "news", note: "氣象預警的固定句式。" },
    { jp: "円安[えんやす]がさらに進[すす]む恐[おそ]れがあると専門家[せんもんか]は指摘[してき]する。", zh: "專家指出日圓恐進一步貶值。", source: "news" },
    { jp: "個人情報[こじんじょうほう]が流出[りゅうしゅつ]した恐[おそ]れがある。", zh: "個資恐已外洩。", source: "news", note: "た形＋恐れ＝「可能已經發生」。" }
  ],
  exercises: [
    { type: "choice", q: "下列哪個情境「不能」用「恐れがある」？",
      options: ["地震で津波が発生する", "試験に合格する", "会社が倒産する", "電車が止まる"],
      answer: 1, explain: "「恐れがある」只用於負面事態，「合格」是好事。" },
    { type: "choice", q: "新聞標題「停電[ていでん]の恐[おそ]れ」的意思是？",
      options: ["已經停電了", "可能會停電", "不會停電", "停電很可怕"],
      answer: 1, explain: "標題名詞止め＝「恐れがある」的縮寫，表可能性。" }
  ]
}
```

## 品質自查（agent 交件前必檢）

- [ ] 檔案第一行是 `window.GRAMMAR_MODULES = window.GRAMMAR_MODULES || [];`，無 markdown fence
- [ ] `node --check <檔案>` 語法通過（心算檢查引號、逗號、括號配對）
- [ ] 所有日文（例句、visual、練習題）漢字都有 `漢字[かな]` 標記
- [ ] 每點 ≥3 例句（過半是 news/ad）、≥2 練習題、有 visual
- [ ] level 標記正確；explanation 是台灣繁中、閱讀問題導向
- [ ] 模組有 openingPuzzle、closingDecode、finalExercises（3-6題）
